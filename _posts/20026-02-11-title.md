# Πώς επιβίωσα από το πρώτο μου Pull Request! 🚀

Γεια σε όλους! Σήμερα είναι μια μεγάλη μέρα. Αποφάσισα να σταματήσω να διαβάζω απλώς θεωρία και να "λερώσω" τα χέρια μου με τον κώδικα στο GitHub.

## Τι έκανα σήμερα
Ξεκίνησα το μάθημα **"Introduction to GitHub"** και έμαθα (με την τρίτη προσπάθεια, η αλήθεια να λέγεται!) πώς να ανοίγω ένα Pull Request σωστά. Το GitHub Actions μου έβγαλε την πίστη με τα permissions, αλλά το "πράσινο checkmark" στο τέλος ήταν η καλύτερη ανταμοιβή.

### Τι έμαθα στο ταξίδι:
* **Licenses:** Ένα `LICENSE` στο repo είναι απαραίτητο αν θέλεις να σε πάρουν σοβαρά οι άλλοι developers.
* **Branches:** Μην δουλεύεις ποτέ απευθείας στο `main`. Φτιάξε ένα branch, δοκίμασε και μετά κάνε merge.
* **Automation:** Το GitHub Actions είναι σαν ένας ψηφιακός βοηθός που ελέγχει τα πάντα για σένα.

## Γιατί React;
Φτιάχνω αυτό το portfolio με **React** γιατί θέλω να καταλάβω πώς τα components συνεργάζονται μεταξύ τους. Δεν είναι απλά ένα site, είναι το δικό μου "lab" όπου πειραματίζομαι με:
1. Σύγχρονο UI/UX.
2. Deployment μέσω GitHub Pages.
3. Σύνδεση με APIs.

> "Τα λάθη στο terminal δεν είναι εχθροί, είναι απλά οδηγίες για το τι να φτιάξεις μετά."

Επόμενος στόχος; Να προσθέσω ένα **Dark Mode** στο portfolio μου και να καθαρίσω λίγο το CSS μου!
